2.4.1 生日问题
  • 在生日问题中, 一般对于闰月的处理如下:
    • 如果是闰月, 那么返回2月29日
    • 如果不是闰月, 那么返回3月1日

    1. 子查询a用来计算每位员工的出生日期与当前日期相差的年份, 以及当前日期.
    2. 要计算某员工最近的生日, 需要在birthday列加上diff列的年数. 如果结果大于当前日期, 则年龄需要再加一年
    3. 子查询b增加的两列即cur和next, 这两列分别用于表示今年和明年的生日.
      • 注意: 如果出生日期是2月29日, 且目标日期不是闰年, 那么这两列所包含的将是2月28日,而不是3月1日
    4. 子查询c用来处理闰年问题.
      • 如果出生的日期是闰年2月29日,并且当前年份不是闰年,日期会为2月28日
      • 根据出生日期是29并且目标生日是28日则代表着发生了闰年日期问题, 需要加一天达到3月1日
    5. 判断今年的生日是否已经过了,如果是,则返回下一年的生日, 得到最后的查询结果
