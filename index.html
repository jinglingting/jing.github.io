实际开发中的用到的装饰器函数

  • flask中的@app路由装饰器

    • route装饰器接收参数,和上面接收参数的例子效果类似

    • 接收参数的装饰器应该有外,中间,内三个函数,但是在路由中不需要在每次调用视图函数时做额外动作,只是一次性的注册路由,所有没有了内层函数,直接返回了视图函数本身

      • 每次调用视图函数时实际执行的就是inner函数,但是每次都添加路由规则显然不对

  • django中的login_required装饰器

    • login_required只是将视图函数带了进来,而真实的装饰器函数时actual_decorator
    • actual_decorator是外部装饰器函数,使用了自定的参数,然后返回中间函数
    • 中间函数接收视图函数参数,然后返回内层函数
    • 内层函数根据测试条件执行视图函数或者执行其他指令
    • 内层函数也使用了functools包的wraps装饰器函数来保持被装饰函数的文档字符串等内容

Posted on 2019-03-02 | Edited on 2019-03-10 | In MySQL优化

存储引擎为MyISAM, 定期碎片整理

  • 对表存储引擎为MyISAM进行整理, 使用 optimize table table_name;
  • 对表存储引擎为InnoDB进行整理, 使用 alter table table_name engine=innodb;
    • 对于存储引擎为InnoDb的表整理, 也可以使用 optimize table table_name, 只是会映射为alter table ...的命令
